# 2026-03-18 — Screenshot Macro Extraction Feature

## Changes

- **New integration**: `src/app/lib/integrations/anthropic/index.ts` — lazy singleton Anthropic client
- **New service**: `src/app/lib/services/macros/macros.service.ts` — `extractMacrosFromImage()` calls Claude Haiku vision, validates response with Zod
- **New module**: `src/app/lib/modules/macros/index.ts` — re-exports from macros service
- **New API route**: `src/app/api/macros/extract/route.ts` — POST endpoint, auth-gated, validates media type, returns `{ fat_g, carbs_g, protein_g }`
- **Modified**: `src/app/(app)/page.tsx` — added upload button (Camera icon) between weight field and macro inputs; on file select, reads as base64, calls extract API, auto-fills macro fields
- **Modified**: `.env.example` — added `ANTHROPIC_API_KEY` placeholder
- **Installed**: `@anthropic-ai/sdk`

## Decisions

- Used `claude-haiku-4-5-20251001` for cost efficiency (screenshot extraction is simple)
- Lazy-initialized Anthropic client to avoid build-time env var errors
- Upload button uses hidden file input pattern with `accept="image/*"`
- User can still manually edit auto-filled values before saving
