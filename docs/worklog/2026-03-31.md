# 2026-03-31

## Session 1: Leaf Burst, Creature Reveal System, Plans

### Woodland save celebration enhancements
- Added leaf emoji projectile burst animation to rustle/leaves phase (`woodland-reveal.tsx`)
  - 3 staggered bursts of 5 emojis each, launched from rustle text positions
  - `requestAnimationFrame` loop with manual projectile physics (gravity, spin, fade)
  - Emoji pool: 🍃🍂🌿🍁🌾🪶🌰
- Rustle text improvements: `text-base` → `text-lg`, opacity 0.7 → 1.0, duration 0.8s → 1.2s
- Documented the save celebration as a user story (`docs/user-stories/woodland-save-celebration.md`)

### Creature reveal system (full-stack)
- New `reveal_odds` table — probability weights per sprite animation (migration 007)
- New `reveal_log` audit table — tracks every encounter with `first_encounter` flag
- New `reveal` domain: service (`rollReveal`, `logEncounter`), module, API routes
  - `POST /api/reveal/roll` — server-side probability roll (prefetched during pawprints)
  - `POST /api/reveal/log` — fire-and-forget audit (only when creature is actually shown)
  - `GET+PUT /api/admin/reveal` — admin odds management with Zod validation
- Admin reveal page (`/admin/reveal`) with live probability distribution bar, sprite previews, weight inputs
- New `"reveal"` phase in woodland state machine: `idle → pawprints → leaves → reveal → idle`
- `CreatureReveal` overlay: spring-animated sprite entrance, "New" badge on first encounters, tap-to-dismiss
- Prefetched roll: API call fires during pawprints phase, result consumed after leaves

### Key decisions
- Server-side roll to prevent client odds manipulation
- Separate `reveal_odds` table (not a column on `sprite_animations`) — not all animations are reveal candidates
- Split roll vs audit: prefetch during pawprints, audit only when user sees the creature
- `requestAnimationFrame` + manual physics for leaf burst — zero new dependencies
