## Session 3: Overscroll Creature ‚Üí Woodland Reveal

- Built `OverscrollCreature` ‚Äî random creature (35% chance) on woodland background below page content
- Iterated on positioning: creature clipped by nav bar, background not flush, gap issues
- Upgraded to `WoodlandReveal` with Framer Motion `useScroll` + `useTransform` for scroll-driven parallax
- **Hit a fundamental design conflict**: scroll runway adds page height ‚Üí content scrolls up. Apple-style parallax works when the entire page is a scroll experience, but breaks when there's a functional form that should stay put.
- Leaf positioning broken (all cluster top-left) ‚Äî `%` values in fixed container not distributing correctly
- **Status: WIP, needs rethink**. The interaction model needs to change ‚Äî can't use scroll distance to drive animation without displacing the form.

### Possible alternative approaches to explore:
- Touch gesture detection (swipe up from bottom edge) instead of scroll
- Bottom sheet / drawer pattern that slides up over the content
- IntersectionObserver on the bottom of the form (no extra scroll distance, just detect when bottom is visible)
- CSS `overscroll-behavior` + rubber-band detection on iOS
